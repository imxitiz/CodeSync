# Plan: Move backend state to Redis (GitHub Issue #11)

**Repo:** `imxitiz/CodeSync` · **Issue:** [#11](https://github.com/imxitiz/CodeSync/issues/11) · **Branch:** `ts-conversion`

## Goal

Replace the 6 in-memory `Map`s in `codesync-server/src/socket.ts` with a Redis-backed **Repository layer** (storage-only refactor). Socket handlers' control flow stays intact; the frontend event contract is **untouched**. Room state survives server restarts and is ready for horizontal scaling.

**Decisions (locked with user):**
- Library: `ioredis`
- Scope: Storage layer only (no separate Service layer)
- Redis required always (no in-memory fallback)
- Room lifecycle: TTL (24h) safety net **+** explicit delete on last-user-leave (preserves current 500ms cleanup)

---

## Current State (the problem)

All room/user state lives in 6 in-memory `Map`s inside `setupSocket()` at `codesync-server/src/socket.ts:51-56`:

| Map | Key → Value | Lines |
|---|---|---|
| `userSocketMap` | socketId → username | 51 |
| `roomCreatorMap` | roomId → creatorUsername | 52 |
| `roomTabsMap` | roomId → Map<tabId, {name, code}> | 53 |
| `userActiveTabMap` | socketId → tabId | 54 |
| `roomCurrentEditorMap` | roomId → editorUsername \| "" | 55 |
| `roomPermissionsMap` | roomId → Map<username, UserPermissions> | 56 |

11 inline handlers (lines 114-425) read/write these directly. **All lost on restart.**

## Redis Key Schema

Pattern: `codesync:<scope>:<id>[:<sub>]`. Room keys get 86400s (24h) TTL, refreshed on activity.

| Key | Type | Value |
|---|---|---|
| `codesync:socket:{socketId}:username` | STRING | username |
| `codesync:socket:{socketId}:activeTab` | STRING | tabId |
| `codesync:room:{roomId}:creator` | STRING | creator username |
| `codesync:room:{roomId}:currentEditor` | STRING | editor username \| "" |
| `codesync:room:{roomId}:tabs` | HASH | field=tabId → JSON `{name, code}` |
| `codesync:room:{roomId}:permissions` | HASH | field=username → JSON `UserPermissions` |
| `codesync:room:{roomId}:usernames` | SET | usernames in room (atomic duplicate check) |

**HASH for tabs/permissions** → single `HGETALL` for `TAB_SYNC` snapshot, atomic `HSET`/`HDEL` per tab/user.
**`usernames` SET** → `SADD` return value gives atomic duplicate-user check (works across future instances).

## Repository Interface (`db/repository.ts`)

One `RoomRepository` class (all state is room-scoped, tightly coupled). Every method is `async`, <50 lines.

**Per-socket:**
- `setSocketUser(sid, username)` / `getSocketUser(sid)` / `deleteSocketUser(sid)`
- `setSocketActiveTab(sid, tabId)` / `getSocketActiveTab(sid)` / `deleteSocketActiveTab(sid)`
- `getSocketUsers(sids[])` / `getSocketActiveTabs(sids[])` — pipeline batch reads

**Room creator (atomic init):**
- `tryInitRoom(roomId, creator)` → `SETNX creator` + `MULTI/EXEC` to init tabs/permissions/currentEditor/usernames. Returns `true` if this caller is the creator.

**Room current editor:**
- `setRoomCurrentEditor(rid, editor)` / `getRoomCurrentEditor(rid)` (returns `""` if missing) / `deleteRoomCurrentEditor(rid)`

**Room tabs (HASH):**
- `setRoomTabDefault(rid)` → `HSETNX tabs tab-main {name:"main.js",code:""}` (idempotent)
- `getAllRoomTabs(rid)` → `HGETALL` → `Map<tabId, TabData>`
- `getRoomTab(rid, tabId)` / `setTabCode(rid, tabId, code)` / `createTab(rid, tabId, name)`
- `deleteTabIfMultipleTabs(rid, tabId)` → **Lua script** (`if HLEN>1 then HDEL; return 1 else return 0`) — fixes last-tab race
- `renameTab(rid, tabId, name)` / `getRoomTabCount(rid)`

**Room permissions (HASH):**
- `setUserPermission(rid, username, perms)` / `setUserPermissionIfNotExists(rid, username, perms)` (`HSETNX`)
- `getUserPermission(rid, username)` / `getAllRoomPermissions(rid)`

**Room usernames SET:**
- `addRoomUsername(rid, username)` → `SADD` returns 1 (added) or 0 (duplicate)
- `removeRoomUsername(rid, username)` / `getRoomUsernames(rid)`

**Bulk / lifecycle:**
- `deleteRoomState(rid)` → pipeline `DEL` of all 5 room keys
- `deleteSocketState(sid)` → pipeline `DEL` of 2 socket keys
- `refreshRoomTTL(rid)` → pipeline `EXPIRE 86400` on all 5 room keys
- `ping()` → for health endpoint

**Serialization:** `JSON.stringify/parse` of `TabData` and `UserPermissions` is contained inside repo methods — handlers never see JSON.

## Concurrency Strategy

| Risk | Where | Fix |
|---|---|---|
| Duplicate user | JOIN | `SADD usernames` return value (atomic) |
| Room init race | JOIN | `SETNX creator` + `MULTI/EXEC` in `tryInitRoom` |
| Last-tab deletion | TAB_CLOSE | Lua script `deleteTabIfMultipleTabs` |
| Everything else | — | Acceptable race window, no transactions needed |

---

## Handler Changes (socket.ts)

All 11 handlers become `async`. Each replaces `Map.get/set/has/delete` with `await repo.*` calls. Emit payloads stay **identical** (frontend contract preserved).

Key transformations:
- **JOIN** → `addRoomUsername` (dup check) → write per-socket state → `tryInitRoom` (atomic) → set perms if not creator → batch-read state → emit `JOINED` + `TAB_SYNC` → `refreshRoomTTL`
- **CODE_CHANGE** → `Promise.all([getRoomCreator, getUserPermission, getRoomCurrentEditor])` → `setTabCode` → `refreshRoomTTL` → broadcast
- **SYNC_CODE** → **unchanged** (only reads Socket.IO internals, no Maps)
- **TAB_CODE_REQUEST** → `getRoomTab` → emit `TAB_CODE`
- **SET_CURRENT_EDITOR** → batch read creator + current editor → `setRoomCurrentEditor` → broadcast
- **TAB_CREATE/RENAME** → perm check → `createTab`/`renameTab` → `refreshRoomTTL` → broadcast
- **TAB_CLOSE** → perm check → `deleteTabIfMultipleTabs` (Lua) → broadcast if deleted
- **TAB_SWITCH** → `setSocketActiveTab` → broadcast
- **PERMISSIONS_UPDATE** → owner check → `setUserPermission` → `refreshRoomTTL` → broadcast
- **disconnecting** → emit `DISCONNECTED` per room → schedule 500ms cleanup (`deleteRoomState` if room empty) → clear current editor if leaver was editor → `deleteSocketState` + `removeRoomUsername`

**Helpers** (`getAllConnectedClients`, `getUserActiveTabs`) become async — still enumerate via `io.sockets.adapter.rooms` (source of truth for socket lifecycle), but batch-read username/activeTab from Redis via pipeline. Keeps path to horizontal scaling open (`@socket.io/redis-adapter` later makes adapter cross-instance).

**Ordering rule:** persist to Redis BEFORE emitting to room (emit-first was safe with sync Maps; with async Redis, await writes first).

---

## Room Cleanup Flow

```
disconnecting
  ├─ emit DISCONNECTED per room
  ├─ for each room: setTimeout(500ms)
  │    └─ if io.sockets.adapter.rooms.get(roomId) is empty
  │         → deleteRoomState(roomId)  [explicit delete]
  ├─ clear currentEditor if leaver was editor → emit SET_CURRENT_EDITOR ""
  └─ deleteSocketState(sid) + removeRoomUsername(room, username)
```

TTL safety net: every mutable handler calls `refreshRoomTTL` → `EXPIRE 86400` on all 5 room keys. Inactive rooms expire naturally even if explicit delete is missed (server crash mid-disconnect).

- **Verify:** `docker compose up -d redis` → `redis-cli ping` → PONG. `bun run dev` starts clean.

### Phase 1 — Repository implementation (group by group)
Implement methods in order: per-socket → room creator → tabs → permissions → usernames → bulk ops.
- **Verify each:** `redis-cli MONITOR` in one terminal, backend in another, observe commands. Inspect with `HGETALL`/`SMEMBERS`.

### Phase 2 — Wire handlers (one at a time, E2E test after each)
Order (simplest → most complex): `TAB_CODE_REQUEST` → `TAB_SWITCH` → `TAB_CREATE`/`TAB_RENAME` → `TAB_CLOSE` (Lua) → `CODE_CHANGE` → `PERMISSIONS_UPDATE` → `SET_CURRENT_EDITOR` → `JOIN` → `disconnecting`.
- **Verify after each:** open 2 browser tabs, join same room, exercise that handler's feature.

### Phase 3 — Cleanup + TTL
Wire `refreshRoomTTL` into mutable handlers; wire `deleteRoomState`/`deleteSocketState`/`removeRoomUsername` into `disconnecting`; wire `addRoomUsername` into `JOIN`.
- **Verify:** restart server mid-session → rejoin → room state persists.

## Risks & Mitigations

| Risk | Mitigation |
|---|---|
| `exactOptionalPropertyTypes` + ioredis options | Use `new Redis(url)` string arg; if options object needed, never set props to `undefined` |
| JSON round-trip on every CODE_CHANGE | ~1µs, negligible. Optimize later only if profiling flags it |
| Socket.IO doesn't await async handlers | Fine — Redis commands are atomic; per-socket events stay ordered by engine.io |
| TTL refresh cost on CODE_CHANGE (5 EXPIREs) | Pipeline = 1 round-trip; Redis handles millions ops/sec — negligible |
| `usernames` SET leak on partial JOIN failure | 24h TTL self-heals; acceptable for MVP |
| Cleanup race (rejoin during 500ms window) | Microsecond window; `SETNX` in `tryInitRoom` makes re-init safe; accept for MVP |
| Health endpoint says OK when Redis down | Add `repo.ping()` check to `/api/health` → return 503 if Redis down |
| No server `type-check` script | Add `"type-check": "bunx tsc --noEmit"` to `codesync-server/package.json` |
